# How to Contribute to Atlas

Thanks for wanting to help with Atlas! This guide will show you how to report bugs, suggest features, and send code changes.

## Reporting Bugs

If you find a bug, please open an issue on GitHub with:

- A clear title
- Steps to make the bug happen
- What you expected and what actually happened
- Your computer info (OS, Python version, browser)
- Screenshots or error messages if you can

## Suggesting Features

Want a new feature? Open an issue and tell us:

- What the feature is
- Why it would be useful
- Any ideas on how to build it

## Sending Code Changes (Pull Requests)

1. Fork the repo and make a new branch:

```bash
git checkout -b feature/your-feature-name
```

2. Make your changes and follow the code style.
3. Test your changes on your computer.
4. Commit your changes with a clear message:

```bash
git commit -m "Add feature: description"
```

5. Push your branch:

```bash
git push origin feature/your-feature-name
```

6. Open a pull request on GitHub and explain what you did.

## Code Style

### Python

- Follow PEP 8 style
- Use clear names
- Write docstrings for functions and classes
- Use type hints when you can

### JavaScript

- Use consistent spaces or tabs
- Use modern JavaScript (ES6+)
- Comment tricky parts
- Keep code small and reusable

## Testing

Before sending your code:

- Make sure the backend starts without errors
- Test the frontend in different browsers
- Check that old features still work
